# Manual de Buenas Prácticas para Trabajos Universitarios en Grupo

Este repositorio contiene un manual colaborativo que recopila buenas prácticas para la organización, planificación y elaboración de trabajos académicos en equipo.
El objetivo principal es practicar el uso de Git y GitHub como herramientas de control de versiones, documentación y trabajo colaborativo, incluso en proyectos NO técnicos.

## Objetivos del proyecto

- Fomentar el uso de Git y GitHub en proyectos académicos.
- Aprender a colaborar efectivamente mediante ramas, issues y pull requests.
- Crear un documento profesional con lineamientos aplicables a cualquier trabajo en grupo.
- Desarrollar habilidades de comunicación, revisión y organización.

## Estructura del repositorio
/documentos        → Contiene el manual y versiones finales en PDF.
/imagenes          → Recursos visuales usados en el manual.
/referencias       → Bibliografía, citas y documentos externos.
/CHANGELOG.md      → Registro de cambios del manual.
/README.md         → Información general del proyecto.

## Flujo de trabajo sugerido

Crear un Issue para cada sección del manual.
Desarrollar cada sección en una rama independiente.
Subir cambios mediante git push.
Abrir un Pull Request y solicitar revisión.
Corregir según comentarios.
Fusionar con la rama main.
Registrar cambios importantes en el CHANGELOG.md.
Versionado:
Este proyecto utiliza versionado semántico simple:
v1.0 → Primera versión completa del manual.
v1.1+ → Correcciones, mejoras o nuevas secciones.
Cada actualización del manual debe ir acompañada de un tag y una entrada en el CHANGELOG.md.

## Entrega final

Este repositorio contiene:

Issues creados para cada sección.
Ramas correspondientes al desarrollo del manual.
Pull Requests asociados.
